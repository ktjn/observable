# UI-R3: Remove Remaining Legacy Style Drift and Document Frontend Migration Rule

**Date:** 2026-04-29
**Status:** Approved
**Phase:** P3 UI renovation gate
**Plan ref:** `docs/superpowers/plans/2026-04-18-phases2-8-iteration-plan.md` — UI-R3

---

## Goal

After UI-R1 and UI-R2, a residual set of legacy CSS classes and inline style objects remains
across product pages. This slice removes them, adds the three primitives needed to cover the
high-frequency patterns, and writes the migration rule that all future frontend slices must follow.

---

## New Primitives

Three new files added to `apps/frontend/src/components/ui/`:

### `loading-state.tsx`

A centered muted-text placeholder rendered while async data is in flight.

```tsx
<LoadingState>Loading service logs…</LoadingState>
```

- Props: `children: React.ReactNode`
- Replaces `<div className="loading-state">` (10+ usages across pages)
- One focused RTL test: renders children in the expected muted style

### `table-panel.tsx`

A scrollable surface wrapper for `<table>` elements.

```tsx
<TablePanel aria-label="Service traces">
  <table>…</table>
</TablePanel>
```

- Props: `children: React.ReactNode`, optional `aria-label: string`, optional `className: string`
- Replaces `<div className="table-panel">` in LogSearch, TraceSearch, ServiceDetailPage (×3), ServiceOverview
- The inner `<table>`, `<th>`, `<td>` element resets remain in `styles.css` — they are global resets, not legacy drift
- One focused RTL test: renders children, forwards aria-label

### `HealthDot` export in `badge.tsx`

A small colored status indicator for health state.

```tsx
<HealthDot state="good" />   // green dot
<HealthDot state="warn" />   // amber dot
<HealthDot state="bad" />    // red dot
<HealthDot state="unknown" /> // muted dot
```

- Props: `state: "good" | "warn" | "bad" | "unknown"`
- Replaces `.entity-health-dot.good/.warn/.bad` in ServiceInfraPanel and `.status.good/.warn/.bad` spans in InfrastructureInventoryPage
- Added as a named export alongside the existing `Badge` component
- One focused RTL test: renders correct token class for each state

## Page Migrations

Every change is a like-for-like style replacement. No behavior, routing, query, or API contract changes.

| File | Legacy removed | Replacement |
|------|---------------|-------------|
| `InfrastructureInventoryPage.tsx` | `metric-tile`, `status .good/.warn/.bad`, `metric-grid`, `loading-state` | `MetricCard`, `HealthDot`, Tailwind grid (`grid grid-cols-4`), `LoadingState` |
| `InfrastructureDetailPage.tsx` | `entry-link-grid`, `entry-link`, `loading-state`, `field-label` (eyebrow) | Tailwind in-place, `LoadingState` |
| `ServiceInfraPanel.tsx` | `entity-card-list`, `entity-card-row`, `entity-card-link`, `entity-card-metric`, `loading-state` | Tailwind in-place, `LoadingState` |
| `ServiceDetailPage.tsx` | `table-panel`, `metric-grid`, `entry-link-grid`, `entry-link`, `loading-state` | `TablePanel`, Tailwind grid, Tailwind in-place, `LoadingState` |
| `ServiceOverview.tsx` | `loading-state`, inline styles on topology controls | `LoadingState`; topology SVG/canvas inline styles quarantined (see Rendering Exceptions) |
| `SetupPage.tsx` | `detail-panel`, `detail-panel-header` | `Panel` primitive |
| `LogSearch.tsx` | `table-panel`, `loading-state` | `TablePanel`, `LoadingState` |
| `TraceSearch.tsx` | `table-panel`, `loading-state` | `TablePanel`, `LoadingState` |
| `AlertsPage.tsx` | `field-label` on form labels | Tailwind: `text-xs font-bold uppercase text-[var(--muted)]` |
| `ProductAreaPage.tsx` | `metric-grid`, `loading-state`, `field-label` (eyebrow) | Tailwind grid, `LoadingState` |

**One-off patterns** (`entry-link`, `entity-card-*`) are used in 1–2 locations. They do not merit
standalone primitives. Replace with Tailwind class strings applied directly in the component.

Remove after all usages are migrated:

- `.metric-tile` and variants (`.good`, `.warn`, `.bad`, `.metric-value`)
- `.empty-panel`, `.empty-title`, `.empty-metrics`
- `.detail-panel`, `.detail-panel-header`
- `.entry-link`, `.entry-link-grid`
- `.entity-card-list`, `.entity-card-row`, `.entity-card-link`, `.entity-card-metric`
- `.signal-panel`
- `.tab-link`, `.tab-list`
- `.loading-state`
- `.status` and variants

**Keep:** `.field-label` and `.metric-label` — used internally by the `Panel` and `MetricCard`
primitives. Add a comment: `/* primitive-internal only — do not use in product pages */`.

## Rendering Exceptions (Approved)

The following inline `style` props are intentional and must not be removed:

1. **Span bar position/width** in `TraceDetail.tsx` — percentage-based `left`/`width` offsets for
   canvas-accurate waterfall rendering. Documented in UI-R2.

2. **Topology SVG/canvas layout** in `ServiceOverview.tsx` — dynamic `x`/`y`/`width`/`height`
   values for force-directed graph node positioning. No Tailwind equivalent.

3. **Severity color styles** in `LogContextView.tsx`, `LogCorrelatedList.tsx`, `LogLiveTail.tsx` —
   dynamically computed numeric severity → color mapping via `getSeverityColor()` /
   `severityColor()`. No static Tailwind class can substitute a runtime-computed color value.

## Migration Rule (added to `spec/15-frontend-local-dev.md`)

The following rule is added to the frontend local dev spec as the authoritative guide for all
future frontend slices:

> **Frontend styling rule:** Reuse `components/ui` primitives first. If a needed primitive does
> not exist, add it in the same slice with a focused RTL test. Avoid page-local interactive
> styles — use Tailwind token classes (e.g. `text-[var(--muted)]`, `bg-[var(--surface)]`).
> Approved rendering exceptions (span bar offsets, topology canvas layout, dynamic severity
> colors) may use inline `style` props; all others require reviewer justification.
